# 承認済み仕様に基づく実装

`spec.md` を**唯一の仕様正**として実装してください。ビルトイン implement と同様に、テスト・ビルド・レポート出力を行います。

Piece Context の Report Directory 内のファイルのみを一次情報として参照してください。

**重要:**

- `spec.md` と矛盾する実装はしない。仕様に穴がある場合はレポートに残し、最小限の補足決定は `coder-decisions.md` に記録する
- 新規コードは既存パターンに合わせる（近傍の実装を読んでから手を入れる）
- 単体テストの追加・更新、ビルド、テスト実行を完了してから完了扱いにする

**出力契約:** `implement.md` に準ずる（変更スコープ宣言・決定ログ・自己チェック）
