# 受入（Acceptance）

`spec.md` の受入条件と、実装結果（コード・テスト・レポート）の整合を確認してください。

**参照（優先順）:**

1. `spec.md`
2. `coder-scope.md` / `coder-decisions.md`（存在する場合）
3. 実際のソースファイル

**やること:**

1. `spec.md` から検証可能な受入条件を列挙する
2. 各条件についてコードまたはテスト・実行証跡に根拠を付けて ✅/❌ を付ける
3. スコープクリープや部分実装がないか確認する
4. ❌ またはブロッキング指摘が残る場合は、修正依頼として具体化する

**出力:** `acceptance.md`（output contract どおり）

**分類（rules 用・サマリーに明示）:**

- 承認する場合は「受入承認」と書く
- 差し戻しの場合は「修正が必要」とし、修正依頼を具体化する
