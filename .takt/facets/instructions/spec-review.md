# 仕様レビュー

`spec.md` を一次情報としてレビューし、実装フェーズに進めるか判断してください。

**参照:**

- `spec.md`
- 必要に応じて Report Directory の他レポート
- 前回のレビュー指摘がある場合は Previous Response を参照し、解消状況を追跡する

**判定方針:**

- **ブロッキング**: 要件の欠落、矛盾、検証不可能な曖昧さ、スコープ外の重大な仮定
- **非ブロッキング**: 表現の改善、実装後でもよい細目

**出力:** `spec-review.md`（output contract どおり）

**分類（rules 用・サマリーに明示）:**

- `APPROVE` の場合は「仕様が承認され実装に進める」と書き、根拠を述べる
- `NEEDS_REVISION` の場合は「仕様の修正が必要」とし、修正指示を列挙する
