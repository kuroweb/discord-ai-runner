# リリース前監督

受理ループを終えた状態で、**リリース（ドキュメント更新前段）に進んでよいか**を判定してください。

**参照:**

- `spec.md`
- `acceptance.md` および関連レポート
- Report Directory 内の全レポートの整合（仕様→実装→受入の筋が通るか）

**やること:**

1. 未解消のブロッキング事項がないか確認する
2. 次の**いずれか一つだけ**をレポート先頭の「判定」に明記する（ピースの遷移条件と一致させる）:
   - `ALL OK（リリース準備完了）` — ドキュメント更新に進んでよい
   - `仕様のフォローが必要` — 合意・仕様の見直しが先。実装だけでは解けない
   - `実装・品質のフォローが必要` — コード・テスト・品質の修正で完結しうる

**出力:** `supervise-release.md`（output contract どおり）
