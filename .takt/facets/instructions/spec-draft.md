# 仕様ドラフト

ユーザー依頼を、実装者が迷わない**承認可能な仕様書**にまとめてください。

**前提:**

- コード変更は行わない（調査のための読み取りのみ可）
- 成果はレポート `spec.md` に集約する

**やること:**

1. 依頼の目的・受入条件・非ゴールを明文化する
2. 既存コードとの関係（変更/新規、影響モジュールの候補）を、読んだファイルパスを根拠に書く
3. 不明点は「仮定」として明示し、リスクがあれば列挙する
4. 受入時に確認すべき観点（テスト・ビルド・手動確認）を列挙する

**出力:** `spec.md`（output contract どおり）
